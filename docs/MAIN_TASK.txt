2.    Агент по подготовке поздравительных сообщений клиентам (Сбер)
Проблематика
Сотрудники Сбербанка ежегодно затрачивают значительное количество рабочего времени на подготовку индивидуальных поздравлений представителям различных компаний и организаций с профессиональными праздниками, днями рождения и памятными датами. В настоящее время процесс полностью ручной, требует дополнительного персонала и увеличивает нагрузку на фонд оплаты труда.
Задание
Ключевая потребность: это не разовая генерация текста, а создание надежного, автоматизированного конвейера, который регулярно, на основе актуальных данных, идентифицирует поводы для поздравлений, генерирует искренне персонализированный контент и доставляет его через правильный канал в нужный момент, постоянно обучаясь на результатах для повышения эффективности. Регулярность здесь — основа процесса, а персонализация — его ценность.
Описание процесса:
1. Триггер события (регулярный/событийный):
Регулярность: система ежедневно/еженедельно проверяет базу данных на предмет наступающих событий:
Дни рождения клиентов (на X дней вперед, напр., 7 дней).
Праздничные даты (Новый год, 8 Марта, профессиональные праздники - если релевантно).     
Событийный: запрос на поздравление может быть инициирован вручную сотрудником (напр., получения гос.награды клиентом)
2. Извлечение данных (Регулярный):
Ежедневно/По триггеру: AI-ассистент запрашивает из базы данных (CRM) информацию о клиенте(ах), для которых сработал триггер:
Обязательный минимум: Имя, Фамилия, тип события (ДР, юбилей, праздник), контакт для отправки (email, номер телефона для SMS/мессенджера).
Вторым этапом для дальнейшей проработки:
Для персонализации: предпочтения (если есть), название компании клиента, должность контактного лица, история взаимодействий (последний контакт, тема), сегмент клиента (VIP, новый, лояльный и т.д.)
3.  Анализ данных и генерация контента (По требованию/По триггеру):
Выбор шаблона/тона: Определяет подходящий шаблон поздравления (официальный, дружеский, креативный) и тон на основе сегмента клиента, истории отношений, типа события.
Персонализация:
Генерация уникальных фраз: Создание коротких, релевантных пожеланий на основе контекста.
Генерация изображения на основе контекста.
4.         Отправка (Регулярный/По триггеру):
По расписанию/Сразу после генерации: Интеграция с каналами коммуникации
Email: Отправка через почтовый сервис.
Предлагаемое решение:
1. **Сбор данных:**
   Мониторинг и обновление информации о датах профессиональных праздников.
2. **Формирование базы организаций:**
   Создание списка ИНН организаций, подлежащих поздравлению.
3. **Извлечение ключевой информации:**
   Автоматическое получение кодов видов экономической деятельности (ОКВЭД) и имен руководителей компаний по каждому ИНН для персонализации поздравлений.
4. **Подготовка поздравлений:**
   Генерация индивидуальных поздравительных текстов и открыток с учётом особенностей деятельности каждой организации.
5. **Отправка поздравлений и оповещение сотрудников:**
   Отправка готовых поздравительных материалов клиентам и уведомления об отправке сотрудникам банка.
Данные
Используются данные сайтов компаний, информационных сайтов, социальных сетей.
Формат решения
           Результатом выполения задания должа быть представлена модель MVP, демонстрацию которого нужно будет провести в конце хакатона, а также в виде презентации, описывающей преимушества, недостатки и пути дальнейшего развития вашего решения.
